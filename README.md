# 2026 Forecast Data

Data files for the MapWise 2026 midterm election forecast. Generated and consumed by [2026_forecast_code](https://github.com/MapWiseResearch/2026_forecast_code).

---

## Directory Structure

```
data/
├── candidates.xlsx              # Master candidate/race database (manual)
├── historical_elections.xlsx    # District and state-level presidential results (2016–2024)
├── national.csv                 # Historical midterm economic indicators (model training data)
├── midterms.csv                 # Historical midterm approval + outcome data (model training data)
├── candidate_fundraising.xlsx   # FEC fundraising data by candidate
├── poll_averages.xlsx           # Weighted poll averages (Senate, House, Governor, Approval, GCB)
└── output/
    ├── preprocessed_data.xlsx   # Merged model-ready data (candidates + fundraising + polls + lean)
    ├── forecast_output.xlsx     # Latest forecast (all races + simulation results)
    ├── nowcast_output.xlsx      # Latest nowcast (polls-only, election-day simulation)
    ├── summary.xlsx             # Cross-run timeline comparison
    └── details/
        └── YYYY-MM-DD/
            ├── forecast_output.xlsx
            └── nowcast_output.xlsx
```

## File Descriptions

### Input / Reference Files

| File | Updated | Description |
|------|---------|-------------|
| `candidates.xlsx` | Manual | Two sheets: `candidates` (one row per candidate with forecast ID, name, composite strength) and `races` (one row per race with district code, Dem/Rep IDs, incumbent party) |
| `historical_elections.xlsx` | Manual | Presidential results by district (`districts` sheet) and state (`states` sheet) for 2016, 2020, 2024 |
| `national.csv` | Static | Historical midterm-year economic indicator values used to train the economic fundamentals regression |
| `midterms.csv` | Static | Historical midterm presidential approval ratings and generic ballot outcomes used to train the approval regression |

### Pipeline-Generated Files

| File | Generated by | Description |
|------|-------------|-------------|
| `candidate_fundraising.xlsx` | `scrape_candidate_fundraising.py` | FEC total raised, spent, cash on hand, debt, PAC/individual/party breakdown — sheets: `Senate`, `House`, `Governor` |
| `poll_averages.xlsx` | `scrape_polls.py` | Time-decayed, bias-adjusted weighted poll averages — sheets: `Averages`, `Generic Ballot`, `Approval`, plus raw sheets per race type |
| `output/preprocessed_data.xlsx` | `merge_forecast_data.py` + `generic_ballot.py` | All races merged with fundraising, election history, partisan lean, poll averages, and national indicators (Generic Ballot + Approval sheets appended by `generic_ballot.py`) |
| `output/forecast_output.xlsx` | `forecast.py` | Full probabilistic forecast — sheets: `All Races`, `Governor`, `Senate`, `House`, `Competitive`, `Ratings`, `State Summary`, `Simulation`, `Summary`, `Generic Ballot`, `Approval` |
| `output/nowcast_output.xlsx` | `forecast.py` | Same structure as forecast but simulated as if election were today (polls-only GCB, days_out=0) |
| `output/summary.xlsx` | `run_all.py` | Cross-run timeline — sheets: `National Timeline`, `Generic Ballot`, `Pres. Approval` |

### Backdated Run Archive (`output/details/`)

Each `YYYY-MM-DD/` subdirectory contains `forecast_output.xlsx` and `nowcast_output.xlsx` for that date's run. Generated by `batch_backdate.py`. Covers 2026-01-01 onward at daily intervals.

---

## Key Identifiers

- **Forecast ID** (`MW-NNNNNN`): MapWise internal candidate identifier, used to link candidates across all files
- **District Code**: Race identifier in the format `ST-NN` (House), `ST-SEN` (Senate), `ST-GOV` (Governor) — e.g. `PA-07`, `PA-SEN`, `PA-GOV`

---

## Running the Pipeline

See [2026_forecast_code](https://github.com/MapWiseResearch/2026_forecast_code) for pipeline usage. To regenerate all output files:

```bash
# Full pipeline (from forecasting/ root)
python3 code/run_all.py

# Backdate historical runs
python3 code/batch_backdate.py --start 2026-01-01 --end 2026-03-29
```
